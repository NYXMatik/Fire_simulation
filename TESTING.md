# Test Documentation

This project uses automated tests to verify the fire-spread simulation model,
its stochastic behavior under fixed seeds, and the expected effect of selected
simulation parameters.

## Test groups

The test suite is split into three files:

- `tests/test_behavioral.py` checks specific simulation behaviors, including
  boundary reflection, terrain-dependent spread order, isotropic spread without
  wind, water barriers, wind bias, and controlled-burn behavior.
- `tests/test_parameters.py` checks parameter sensitivity. It verifies that
  higher ignition probability, higher spread speed, wind direction, and longer
  burnout timing change simulation results in the expected direction.
- `tests/test_stability.py` checks reproducibility and stability. It runs
  scenarios across multiple seeds, verifies that variation remains within
  acceptable bounds, and confirms that the same seed gives identical results.

## Reproducibility

The interactive simulation is stochastic, but the tests pass explicit random
seeds to the simulation functions. This keeps individual scenarios reproducible
while still testing random variation across different seeds.

## Local test command

Run the full test suite locally with:

```bash
python -m pytest
```

To generate local report files:

```bash
python -m pytest --capture=tee-sys --json-report --json-report-file=reports/pytest-report.json --html=reports/pytest-report.html --self-contained-html --junitxml=reports/pytest-junit.xml
python scripts/build_test_report.py --json reports/pytest-report.json --output reports/fire-simulation-test-report.html
```

The generated files are placed in `reports/`, which is ignored by Git.

## GitHub Actions

The repository contains a GitHub Actions workflow at
`.github/workflows/tests.yml`. It runs automatically:

- on every push to `main`
- on pull requests targeting `main`
- manually from the GitHub Actions tab

The workflow installs dependencies from `requirements.txt`, runs `pytest`, and
uploads a `fire-simulation-test-report` artifact.

## Generated artifact

Each successful workflow run uploads an artifact named
`fire-simulation-test-report`. It contains:

- `fire-simulation-test-report.html` - custom report with a summary chart,
  per-test status, duration, and captured output
- `pytest-report.html` - report generated by `pytest-html`
- `pytest-report.json` - structured test data generated by `pytest-json-report`
- `pytest-junit.xml` - JUnit XML report for external tools
- `pytest-output.txt` - complete console output from the test run

For course reports, the most readable file to attach or reference is
`fire-simulation-test-report.html`. The GitHub Actions run page can also be
linked as evidence that the tests were executed in CI.

## Badge status

The README badge shows the latest known status of the `Tests` workflow on the
default branch. When the workflow has not finished yet, GitHub may temporarily
show `no status`. After a completed run, it should change to `passing` or
`failing`.
